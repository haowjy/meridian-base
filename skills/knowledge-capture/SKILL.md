# Knowledge Capture

Understanding that lives only in conversation will be lost to compaction.
When you discover structural insight — module purpose, boundary contracts,
architectural rationale — write it down while context is fresh. The goal is
durable understanding: a reader six months from now should find what they need
without replaying the conversation that produced it.

Three layers exist for captured knowledge: code-colocated docs (.context/),
the project KB, and user-facing docs. Each has different scope and audience.

Load /qi-layer for inline placement rules, /kb-conventions for KB structure,
/post-impl-capture for post-ship documentation workflow.
